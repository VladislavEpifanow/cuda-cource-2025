# Lab 4: Radix Sort — Поразрядная сортировка на CUDA

## Описание
Реализация алгоритма Radix Sort на CUDA для сортировки целых чисел (int32/int64).  
Поддерживает отрицательные числа, сравнивается с `qsort` и `thrust::sort`.

## Требования
- CUDA Toolkit 11.0+
- GPU с compute capability 7.0+
- Linux, GCC 9.0+, Make

## Компиляция и запуск
```bash
make
./radix_sort
```
Автоматически тестируются размеры: 10⁵, 5·10⁵, 10⁶ элементов для int32/int64.

## Алгоритм
```
for bit = 0 to 31 (или 63):
    1. mark_bit: помечаем элементы (бит = 0 → 1, бит = 1 → 0)
       • для знакового бита: инверсия для корректной сортировки отрицательных
    2. multi_block_scan: префиксная сумма (Blelloch, shared memory)
    3. scatter: размещаем элементы по новым позициям
    4. swap буферов ввода/вывода
```
## Особенности реализации
* Parallel Scan: рекурсивный Blelloch scan для обработки массивов > размера блока
* Отрицательные числа: инверсия старшего бита сохраняет порядок при сортировке
* Двубуферизация: чередование d_input/d_output без лишних копирований
* Шаблоны: единая реализация для int32_t и int64_t
## Ограничения
* Только целочисленные типы
* Требуется ~3×N дополнительной памяти (is_zero, scan, блок-суммы)
* Для малых массивов (<10⁴) накладные расходы CUDA могут превышать выгоду
## Замечания
* thrust::sort показывает лучшее время на малых размерах за счёт внутренней оптимизации
* На больших массивах (10⁶) кастомный Radix Sort демонстрирует стабильное ускорение 4–15× относительно CPU
* Производительность зависит от архитектуры GPU и пропускной способности памяти
